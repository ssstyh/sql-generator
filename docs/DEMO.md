# 演示说明

## 演示目标

本 runbook 用于把当前版本稳定地演示为“一个已经完成核心闭环、具备最小自动化门禁、适合继续扩展的 SQL 分析模板生成器”。

演示时只展示已经通过 QA 快速复验的能力，不临场发挥未完成功能。

## 推荐演示顺序

1. 仓库与项目定位
2. 趋势分析
3. 同比 / 环比分析
4. 留存分析
5. 历史记录与 P1 修复结果
6. 当前边界与后续规划

## 推荐展示场景

优先使用仓库内已经固化的示例参数进行演示，避免现场手输导致无关错误。

### 1. 趋势分析

- 展示用户输入表名、日期字段、指标字段和聚合方式
- 生成趋势 SQL
- 展示代码块、复制入口和下载入口

### 2. 同比 / 环比分析

- 切换到对比分析
- 演示方言切换为 `postgresql`
- 生成同比或环比 SQL
- 强调当前支持真实模板，而不是占位文本

### 3. 留存分析

- 切换到留存分析
- 演示留存天数与留存模式
- 生成留存 SQL

### 4. 历史记录与修复结果

- 在同一会话内连续生成两类 SQL
- 点击侧边栏历史记录恢复之前结果
- 口头说明两个已经通过 QA 的 P1 修复点：
  - 成功生成后历史记录即时显示
  - 生成失败后不会残留旧成功结果

## 当前可展示功能

- 趋势分析真实 SQL 生成
- 同比 / 环比真实 SQL 生成
- 留存分析真实 SQL 生成
- `mysql` / `postgresql` 方言切换
- 结果展示、复制与下载
- 历史记录与历史恢复
- GitHub Actions 最小 `pytest` CI 门禁

## 当前不要误演示的内容

- 漏斗分析
- RFM 分析
- 真实数据库连接
- 真实 GitHub Actions 首次线上运行结果
- 浏览器级 E2E
- 正式生产发布或线上可用性承诺

对于漏斗 / RFM，只需说明页面已经明确标识“真实模板仍在接入中”，本轮不对外承诺。

## 建议准备的演示素材

建议提前准备以下素材，但本轮只输出说明文档，不伪造素材文件：

- 1 张首页或主界面截图
- 1 张趋势 / 对比 / 留存生成结果截图
- 1 张历史记录与恢复效果截图
- 1 份约 30 秒的 GIF 或口播脚本

## 30 秒口播参考

可以按以下节奏讲解：

“这是一个面向数据分析场景的 SQL 分析模板生成器。当前已经完成趋势、同比环比和留存三类真实 SQL 生成，并提供 Streamlit 页面、历史记录、复制下载和最小 pytest CI 门禁。QA 已完成快速复验，当前适合做演示级部署和对外交付说明；漏斗与 RFM 仍明确保留在后续扩展范围内。”

## 演示收尾建议

演示最后统一强调三点：

- 当前核心链路已经稳定
- 当前边界表达清楚，没有夸大未完成功能
- 下一步应由项目经理基于现有材料安排最终展示与后续排期
