# 使用说明书

## 这是什么

这是一个“帮你生成分析 SQL”的小工具。

你不需要自己从零写 SQL，只要按页面提示填几个信息，它就会帮你生成：

- 趋势分析 SQL
- 同比 / 环比分析 SQL
- 留存分析 SQL

当前版本还不能直接生成：

- 漏斗分析 SQL
- RFM 分析 SQL

所以你现在可以直接用，但请先把它理解成一个“已经能稳定使用的演示版 / 实用版”，不是完整版全功能平台。

## 适合谁用

如果你属于下面任意一种人，这个工具就适合你：

- 不太会写 SQL，但想先得到一个可改的分析模板
- 会一点 SQL，但不想每次重复写趋势、同比、留存
- 想先快速看到分析 SQL 长什么样，再交给技术同事微调

## 你使用前要准备什么

你至少要知道这些信息里的大部分：

1. 你的表名
2. 你的日期字段名
3. 你的指标字段名
4. 你想看哪种分析
5. 你用的是 `MySQL` 还是 `PostgreSQL`

如果你暂时没有真实数据表，也没关系。
你可以先用页面里的示例参数练习，先学会怎么操作。

## 第一步：打开工具

先进入项目目录，然后运行：

```bash
streamlit run app.py
```

如果是第一次使用，建议先安装依赖：

```bash
pip install -r requirements.txt
```

启动后，浏览器会自动打开页面。

## 第二步：先认识页面

页面里你主要会看到这几块：

### 1. 分析类型

你可以在页面里切换：

- 趋势分析
- 同比 / 环比
- 留存分析

漏斗分析和 RFM 目前会显示“还在接入中”，这两个先不要用。

### 2. SQL 方言

你可以选择：

- `MySQL`
- `PostgreSQL`

如果你不确定自己用哪一个，先问技术同事。
选错了，生成出来的 SQL 可能细节不兼容。

### 3. 参数输入区

这里是你填写业务信息的地方，比如：

- 表名
- 日期字段
- 指标字段
- 聚合方式
- 日期范围
- 筛选条件

### 4. 结果区

生成成功后，这里会显示：

- 生成好的 SQL
- 复制按钮
- 下载按钮
- 参数摘要

### 5. 历史记录

你每生成一次，左侧或侧边栏会保留一条历史记录。
点历史记录，可以把之前的结果重新调出来。

## 第三步：最简单的使用方法

如果你完全不会，最推荐你这样用：

1. 先选一种分析类型
2. 点击“加载示例”
3. 直接点“生成 SQL”
4. 先看生成结果
5. 再慢慢把示例里的表名和字段改成你自己的

这样最稳，也最不容易一上来就报错。

## 三种分析怎么用

## A. 趋势分析

### 适合看什么

适合看一段时间内指标怎么变化，比如：

- 每天订单金额趋势
- 每周新增用户趋势
- 每月付费人数趋势

### 你一般要填什么

- 表名
- 日期字段
- 指标字段
- 聚合方式
- 时间粒度：天 / 周 / 月
- 日期范围
- 可选筛选条件

### 例子

如果你想看“3 月每天订单金额总和”，那你可以这样理解：

- 表名：`user_orders`
- 日期字段：`order_date`
- 指标字段：`amount`
- 聚合方式：`sum`
- 时间粒度：`day`

然后点“生成 SQL”就行。

## B. 同比 / 环比分析

### 适合看什么

适合看“和上一期比怎么样”，比如：

- 这个月比上个月增长多少
- 今年 3 月比去年 3 月增长多少

### 你一般要填什么

- 表名
- 日期字段
- 指标字段
- 聚合方式
- 时间粒度
- 对比类型：同比 / 环比
- 日期范围
- 可选筛选条件

### 什么时候选同比，什么时候选环比

- `同比`：和去年同一个时间比
- `环比`：和上一个周期比

如果你只是想看“这个月比上个月”，一般选环比。

## C. 留存分析

### 适合看什么

适合看用户是否会回来，比如：

- 新用户第 1 天、第 3 天、第 7 天还在不在
- 用户留存曲线怎么样

### 你一般要填什么

- 表名
- 用户 ID 字段
- 日期字段
- 留存天数
- 留存模式
- 开始日期和结束日期
- 可选筛选条件

### 你会看到两种输出思路

- 留存宽表
- 留存曲线

如果你只是第一次用，先用示例跑一遍最容易理解。

## 第四步：如何加筛选条件

筛选条件就是“只看某一部分数据”。

比如你只想看：

- 某个地区
- 某个渠道
- 某个平台

你可以添加筛选条件。

常见操作符有：

- `=`
- `!=`
- `>`
- `>=`
- `<`
- `<=`
- `IN`
- `LIKE`

### 最容易理解的几个写法

#### 1. 只看一个地区

- 字段：`region`
- 操作符：`=`
- 值：`APAC`

#### 2. 同时看多个渠道

- 字段：`channel`
- 操作符：`IN`
- 值：`organic, ads`

注意：
如果你选 `IN`，多个值通常要用英文逗号隔开。

## 第五步：生成后怎么处理

生成成功后，你可以这样做：

### 1. 直接复制

点击复制按钮，把 SQL 粘贴到你的数据库查询工具里。

### 2. 直接下载

如果你想保存下来，点击下载按钮，把 SQL 文件存起来。

### 3. 回看历史结果

如果你刚刚生成过别的分析，不用重填，直接点历史记录恢复。

## 最常见的报错怎么理解

### 1. “表名不能为空”

意思是你没有填表名。

解决方法：
把表名补上再生成。

### 2. 字段名不合法

意思通常是字段名为空，或者格式不符合要求。

解决方法：
检查字段名有没有写错、有没有多余空格。

### 3. 日期范围不对

意思通常是开始时间和结束时间不合理。

解决方法：
重新选择日期范围。

### 4. 留存参数不对

意思通常是留存天数没填好，或者格式不对。

解决方法：
先加载示例，再按示例格式修改。

## 最适合小白的使用顺序

如果你完全没接触过这个工具，建议严格按这个顺序来：

1. 启动页面
2. 先做“趋势分析”
3. 使用示例参数生成一次
4. 把示例里的表名、字段换成你自己的
5. 再试“同比 / 环比”
6. 最后再试“留存分析”

不要一开始就碰：

- 漏斗分析
- RFM
- 真实数据库接入

因为这些还不是当前版本的交付范围。
