# 3 分钟上手版

## 先说结论

这个项目现在已经可以直接用来做三类分析 SQL 生成：

- 趋势分析
- 同比 / 环比分析
- 留存分析

当前不要用它做：

- 漏斗分析
- RFM 分析
- 真实数据库直连

## 1 分钟启动

先在项目目录运行：

```bash
pip install -r requirements.txt
streamlit run app.py
```

浏览器打开后，就能看到页面。

## 30 秒看懂页面

你只要先盯住这 4 个地方：

1. 分析类型
   这里选“趋势分析”“同比 / 环比”“留存分析”
2. SQL 方言
   这里选 `MySQL` 或 `PostgreSQL`
3. 参数输入区
   这里填表名、字段名、日期范围等
4. 生成结果区
   这里看 SQL，支持复制和下载

## 最适合新手的上手方法

不要一开始自己乱填。

最稳的做法是：

1. 选一个分析类型
2. 点击“加载当前分析示例”
3. 点击“生成 SQL”
4. 先看结果
5. 再把示例里的表名和字段改成你自己的

## 三种最常用玩法

### 趋势分析

适合看：

- 每天销售额
- 每周新增用户
- 每月订单数

### 同比 / 环比

适合看：

- 这个月比上个月
- 今年 3 月比去年 3 月

### 留存分析

适合看：

- 新用户第 1 天、第 3 天、第 7 天还有没有回来

## 出结果后你怎么用

生成成功后，你可以：

- 直接复制 SQL
- 直接下载 `.sql` 文件
- 点历史记录找回之前生成过的结果

## 最常见报错怎么处理

### 表名不能为空

把表名补上。

### 字段名不对

检查字段名有没有写错，或者有没有空着。

### 日期范围不对

重新选开始和结束日期。

### 留存参数不对

先加载示例，再照着改。
