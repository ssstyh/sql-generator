# 人员配置总览

## 项目定位

本项目为 `SQL 分析模板生成器`，目标是在 10 天内完成一个可上线演示的 MVP，支持用户输入表名和字段后，自动生成常用分析 SQL，包括：

- 趋势分析
- 同比/环比分析
- 留存分析
- 漏斗分析
- RFM 分析

项目技术栈以 `Python + Streamlit + Jinja2` 为主，交付物为一个可在浏览器运行的 Web 工具。

## 最优人员配置

为了保证 `效率最高` 和 `完成度最完整`，建议采用 `6 人小组`：

1. 项目经理 / 产品负责人
2. 技术负责人 / 后端架构师
3. SQL 模板与分析逻辑工程师
4. 前端工程师（Streamlit）
5. 测试工程师（QA）
6. 部署与文档工程师

## 为什么 6 人最合适

- 该项目的核心风险不在页面复杂度，而在 `SQL 分析逻辑正确性` 和 `多分析模板的稳定生成`。
- 如果人员过少，开发、测试、上线会互相挤压，容易出现功能能跑但逻辑不准的问题。
- 如果人员过多，沟通与交接成本会上升，不利于 10 天 MVP 快速推进。
- 6 人配置刚好覆盖 `需求拆解、架构设计、业务逻辑、前端交互、测试验证、上线包装` 六个关键环节。

## 各岗位职责

### 1. 项目经理 / 产品负责人

负责项目目标拆解、进度推进和最终验收。

职责：

- 制定 Day 1 到 Day 10 的推进计划
- 明确 MVP 边界，避免功能失控
- 为每一天定义任务、交付物和验收标准
- 组织每日进度同步，跟踪阻塞与风险
- 统筹最终演示、项目验收和面试话术整理

### 2. 技术负责人 / 后端架构师

负责整体技术方案、代码结构与核心模块治理。

职责：

- 搭建项目基础骨架与目录结构
- 设计 `core/`、`templates/`、`utils/`、`tests/` 的职责边界
- 统一 SQL 生成器接口、参数校验接口和格式化流程
- 设计 MySQL / PostgreSQL 方言兼容方案
- 审核关键代码，保证后续多人协作时结构不失控

### 3. SQL 模板与分析逻辑工程师

负责所有分析 SQL 的模板设计与业务逻辑正确性。

职责：

- 编写趋势、同比/环比、留存、漏斗、RFM 模板
- 维护 `Jinja2` 模板内容和生成参数映射
- 保证输出 SQL 注释清晰、结构统一、可执行
- 重点保证留存 SQL 和 RFM 评分/标签逻辑准确
- 配合测试修复分析逻辑和边界问题

### 4. 前端工程师（Streamlit）

负责用户交互界面和前后端联调体验。

职责：

- 实现分析类型切换、SQL 方言切换和动态表单
- 完成 SQL 展示、Tab 切换、复制与下载
- 实现历史记录、示例加载、欢迎说明和参数摘要卡片
- 做输入校验提示与错误处理
- 配合 Demo 模式，提升面试展示和初次体验效果

### 5. 测试工程师（QA）

负责项目测试体系、边界场景验证和回归把关。

职责：

- 编写趋势、留存、漏斗、RFM、校验器测试用例
- 覆盖正常场景、异常输入和边界条件
- 验证 SQL 是否可被 `sqlparse` 正常解析
- 输出缺陷清单，推动开发修复并执行回归验证
- 对最终上线版本做功能验收

### 6. 部署与文档工程师

负责项目文档、上线配置和仓库展示包装。

职责：

- 完善 `README.md`、部署文档、使用说明和截图规划
- 维护 `.streamlit/config.toml`、`LICENSE`、`.gitignore`
- 协助整理 GitHub badges、description、topics
- 执行 Streamlit Cloud 部署和上线检查
- 配合录制演示 GIF 和整理展示资料

## 推荐执行流程

### Day 1

- 项目经理明确范围和验收标准
- 技术负责人完成项目初始化
- 部署与文档工程师同步准备依赖说明和基础文档

### Day 2 - Day 4

- 技术负责人维护公共接口和架构
- SQL 工程师实现趋势、同比/环比、留存、漏斗、RFM 模板
- 前端工程师提前搭建界面骨架，降低后期集中联调压力

### Day 5 - Day 6

- 前端工程师主导交互开发与联调
- 技术负责人支持接口适配
- SQL 工程师修正联调中暴露的逻辑问题

### Day 7

- 测试工程师编写完整测试套件并执行全量验证
- 技术负责人和 SQL 工程师集中修复问题

### Day 8

- 部署与文档工程师完善 README、部署文档和仓库材料
- 技术负责人统一做格式化、规范和代码收口

### Day 9

- 完成部署上线、Demo 模式与 GitHub 美化

### Day 10

- 项目经理组织最终验收
- 整理演示材料、面试话术和项目介绍

## 岗位协作关系

- 项目经理负责节奏和结果
- 技术负责人负责技术主线和开发质量
- SQL 工程师负责分析逻辑准确性
- 前端工程师负责交互体验和展示闭环
- 测试工程师负责独立验证与回归
- 部署与文档工程师负责上线与项目包装

## 精简人力方案

如果必须压缩人力，最低可以使用 `4 人方案`：

1. 项目经理
2. 技术负责人兼后端
3. SQL 工程师兼测试
4. 前端工程师兼部署文档

但该方案会明显降低：

- 测试深度
- 交付稳定性
- 文档与展示质量

## 审查结论

建议批准 `6 人方案`，这是当前项目在 `10 天周期内` 最稳妥、最高效、完成度也最完整的组织方式。
