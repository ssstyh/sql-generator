# 员工信息与岗位职责

## 基本信息

- 岗位名称：SQL 模板与分析逻辑工程师
- 目录名称：`03_SQL模板与分析逻辑工程师`
- 岗位定位：负责分析方法论落地、SQL 模板编写与业务逻辑正确性
- 项目阶段归属：Day 2 至 Day 7 核心产出岗位

## 在项目中的核心使命

把项目中最重要的 5 类分析能力准确实现为可生成 SQL 的模板，确保结果不仅能运行，还符合真实数据分析场景和面试解释逻辑。

## 具体职责

### 1. 模板设计与生成逻辑实现

- 编写趋势分析模板
- 编写同比 / 环比分析模板
- 编写留存分析模板
- 编写漏斗分析模板
- 编写 RFM 分析模板

### 2. 业务逻辑保障

- 保证趋势聚合、同比、环比逻辑清晰
- 保证留存分析的首次活跃日、回访匹配与分组统计逻辑正确
- 保证漏斗步骤转化率计算合理
- 保证 RFM 原始值计算、分箱和标签逻辑准确

### 3. 模板规范统一

- 为每条 SQL 增加注释头
- 保证模板参数可读、结构统一、可维护
- 兼容 MySQL / PostgreSQL 两种方言

### 4. 缺陷修复

- 配合 QA 修复边界条件问题
- 修复空筛选、关键字字段名、日期范围缺失等场景问题

## 分阶段任务

- Day 2：完成趋势分析、同比环比 SQL 生成
- Day 3：完成留存分析和留存曲线 SQL
- Day 4：完成漏斗分析和 RFM 模板
- Day 5 - Day 6：配合前端联调参数与输出格式
- Day 7：配合测试修复逻辑缺陷

## 主要协作对象

- 技术负责人 / 后端架构师
- 测试工程师（QA）
- 前端工程师（Streamlit）
- 项目经理 / 产品负责人

## 关键交付标准

- 5 类分析模板全部可生成 SQL
- SQL 输出注释清晰，便于理解和演示
- 关键逻辑特别是留存和 RFM 逻辑可经得起 review
- 支持两种 SQL 方言且输出结构一致
