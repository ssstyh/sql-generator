## 本轮任务目标

为当前已开放的三类分析能力补齐可直接验证的模拟数据包，让老板或新手可以在其他数据库工具或平台中导入这些数据，再把页面生成的 SQL 贴进去验证结果。

## 你这轮要完成什么

### 1. 设计并落地模拟数据包

至少覆盖以下两张核心示例表：

- `user_orders`
- `user_events`

字段设计必须与当前页面示例和模板能力对齐，至少要能支撑：

- 趋势分析
- 同比 / 环比分析
- 留存分析

### 2. 提供“看得懂”和“能直接拿去验证”的材料

建议最小交付包括：

- 原始模拟数据文件，例如 `csv`
- 字段说明或表结构说明
- 至少一份可参考的初始化 SQL 脚本
- 一份简短的新手验证说明

如果你认为更合适，也可以提供多种形式，例如：

- `mock_data/*.csv`
- `mock_data/schema.sql`
- `mock_data/seed.sql`

### 3. 保证与当前示例参数一致

你设计的数据包必须能和当前应用里的示例参数直接对上，避免出现：

- 示例表名存在，但字段对不上
- SQL 能生成，但数据无法验证
- 留存分析字段命名和数据含义不一致

### 4. 补齐外部平台验证说明

新增一份面向老板 / 新手的验证说明文档，说明：

- 数据文件在哪
- 两张表示例字段分别是什么
- 在别的平台里先导入什么
- 用哪个页面示例生成 SQL
- 生成后去哪里执行
- 预期能看到什么现象

说明文档默认面向外部数据库工具或平台，例如：

- Navicat
- DBeaver
- DataGrip
- 本地 MySQL / PostgreSQL
- 其他能导入 CSV 并执行 SQL 的平台

### 5. 如有必要，补充测试

如果这轮引入了新的辅助脚本，请补充对应测试；如果主要是新增数据资产和文档，请至少自行完成一轮导入与执行验证，并在总结中写清楚。
