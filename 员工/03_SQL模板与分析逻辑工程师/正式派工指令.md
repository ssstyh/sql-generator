## 四、本次正式交付任务

### 任务 A：完成趋势分析 SQL
请落地以下能力：
- 基础趋势 SQL
- 同比 SQL
- 环比 SQL

请重点保证：
- 时间粒度支持日 / 周 / 月
- 聚合方式逻辑合理
- 注释头清晰
- 输出结构适合后续前端展示

### 任务 B：完成留存分析 SQL
请落地以下能力：
- 经典 N 日留存表
- 留存曲线数据 SQL

请重点保证：
- 首次活跃日计算正确
- N 天后活跃匹配逻辑正确
- 分组统计和留存率计算可解释
- MySQL / PostgreSQL 兼容清晰

### 任务 C：扩展生成器与校验器
请继续完善：
- [core/generator.py](/E:/sql-generator/core/generator.py)
- [core/validator.py](/E:/sql-generator/core/validator.py)

重点是：
- 让真实模板通过统一入口被调用
- 补齐 Day 2 所需业务参数校验
- 保持接口命名稳定，便于后续前端和 QA 接入

### 任务 D：补齐 Day 2 测试
请补充：
- `tests/test_trend.py`
- `tests/test_retention.py`

测试应至少覆盖：
- 两种 SQL 方言
- 多种时间粒度
- 多种留存天数
- 非法参数和缺失参数

### 任务 E：提交完成总结
任务完成后，必须在你的文件夹下生成：
- [员工/03_SQL模板与分析逻辑工程师/Day2完成总结.md](/E:/sql-generator/员工/03_SQL模板与分析逻辑工程师/Day2完成总结.md)

该完成总结必须写清楚：
- 你改了哪些文件
- 你实现了哪些 SQL 能力
- 你做了哪些验证
- 当前有哪些边界没有做
- 你建议下一步交接给谁以及为什么

## 五、边界要求
本次任务有明确边界，请严格控制：

不要做：
- 不要提前实现漏斗分析和 RFM
- 不要提前做 Day 5 的 Streamlit 完整交互
- 不要把复杂展示逻辑混进模板层
- 不要绕过现有统一入口直接从 UI 读取模板

必须做到：
- SQL 逻辑正确
- 模板结构统一
- 输出带注释头
- 支持两种方言
- 测试可验证

3. `tests/test_trend.py` 与 `tests/test_retention.py` 通过
4. 关键逻辑尤其是留存逻辑经得起 review
5. 已在个人文件夹生成 `Day2完成总结.md`
