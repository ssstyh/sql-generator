# 员工信息与岗位职责

## 基本信息

- 岗位名称：技术负责人 / 后端架构师
- 目录名称：`02_技术负责人_后端架构师`
- 岗位定位：负责系统架构、核心模块设计、接口统一与代码质量收口
- 项目阶段归属：Day 1 至 Day 9 核心负责人

## 在项目中的核心使命

为项目建立稳定、可扩展、便于协作的技术骨架，保证 SQL 生成逻辑、模板引擎、参数校验、格式化流程和前端调用接口能够顺畅衔接。

## 具体职责

### 1. 项目架构设计

- 搭建 `app.py`、`core/`、`templates/`、`utils/`、`tests/` 基础结构
- 设计 `generator.py`、`validator.py`、`formatter.py` 的职责边界
- 统一模块命名、参数风格和文件组织方式

### 2. 核心后端能力建设

- 设计 SQL 生成器的统一调用方式
- 定义 MySQL / PostgreSQL 方言差异处理方案
- 设计输入校验机制和异常处理方式
- 统一 SQL 注释头和格式化输出规范

### 3. 联调与质量控制

- 为前端提供稳定的调用接口
- 支持 SQL 工程师实现模板并合入主架构
- 在测试阶段定位并修复核心问题
- 执行 `black`、`flake8`、类型注解和代码规范收口

## 分阶段任务

- Day 1：完成项目初始化和基础结构设计
- Day 2 - Day 4：支持趋势、留存、漏斗、RFM 模板的核心接入
- Day 5 - Day 6：配合前端完成界面联调
- Day 7：和 QA 一起处理测试暴露问题
- Day 8：完成代码规范、注释、质量优化
- Day 9：支持 demo 模式和部署前收口

## 主要协作对象

- 项目经理 / 产品负责人
- SQL 模板与分析逻辑工程师
- 前端工程师（Streamlit）
- 测试工程师（QA）
- 部署与文档工程师

## 关键交付标准

- 代码结构清晰，便于多人并行开发
- 生成器接口统一，前后端可稳定联调
- 多方言支持方案明确且可维护
- 项目具备稳定的可测试性和可部署性
