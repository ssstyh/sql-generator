收件人：`02_技术负责人_后端架构师`
发起人：项目经理
阶段：Day 1 / Phase 1 项目初始化
优先级：高

## 一、任务背景
当前项目已经完成项目经理阶段的启动工作，已明确 MVP 范围、阶段路线图和 Day 1 验收口径。你现在正式接手 Day 1 的技术落地工作。

本次任务的目标不是提前完成业务功能，而是把项目从“文档说明阶段”推进到“具备可运行骨架和清晰模块边界”的状态，为后续 SQL 模板工程师、前端工程师和测试工程师提供稳定基础。

请优先按以下顺序调用：

1. `system-architecture`
用途：明确模块边界、调用关系、方言兼容位置、后续协作接入方式。

## 四、本次正式交付任务

### 任务 A：输出基础架构说明
请先产出 `architecture.md`，至少明确以下内容：
- `app.py` 的职责
- `core/generator.py` 的职责
- `core/validator.py` 的职责
- `utils/formatter.py` 的职责
- `templates/` 中模板如何统一被加载与调用
- 后续 SQL 工程师与前端工程师如何接入当前结构

### 任务 B：完成项目骨架初始化
请创建并整理以下结构：

```text
sql-generator/
├── app.py
├── requirements.txt
├── README.md
├── config/
│   └── db_config.yaml
├── templates/
│   ├── retention.sql.j2
│   ├── funnel.sql.j2
│   ├── trend.sql.j2
│   ├── rfm.sql.j2
│   └── compare.sql.j2
├── core/
│   ├── __init__.py
│   ├── generator.py
│   ├── validator.py
│   └── db_connector.py
├── utils/
│   ├── __init__.py
│   └── formatter.py
└── tests/
    ├── test_generator.py
    └── test_validator.py
```

### 任务 C：完成最小可运行页面
请在 `app.py` 中实现最小 Streamlit 页面，要求如下：
- 页面标题为“SQL 分析模板生成器”
- 可通过 `streamlit run app.py` 正常打开
- 不提前加入 Day 5 的复杂交互逻辑

### 任务 D：预留统一技术接口
请为后续开发预留清晰接口，但不要提前实现复杂业务：
- 统一的生成器入口
- 基础输入校验入口
- SQL 格式化入口
- MySQL / PostgreSQL 方言兼容的预留位置

### 任务 E：补齐基础启动文档
请更新 `README.md`，至少包含：
- 项目简介
- 本地启动方式
- 目录结构说明
- Day 1 当前完成状态

## 五、边界要求
本次任务有明确边界，请严格控制范围：

不要做：
- 不要提前实现趋势、留存、漏斗、RFM 的完整 SQL 逻辑
- 不要提前开发 Day 5 的完整前端交互
- 不要引入超出 MVP 范围的新框架
- 不要为了未来假设需求把结构设计得过深过重

必须做到：
- 结构清晰
- 命名统一
- 后续员工可直接接手
- 骨架真实可运行

## 六、验收标准
你提交时必须满足以下条件：

1. `streamlit run app.py` 能打开页面
2. `requirements.txt` 至少包含：
   - `streamlit`
   - `jinja2`
   - `sqlparse`
   - `pyyaml`
   - `pyperclip`
3. 项目目录结构与 [plan.md](/E:/sql-generator/plan.md) 一致
4. `architecture.md` 能清楚说明模块边界和后续接入方式
5. 后续 SQL 工程师无需重构骨架即可直接进入 Day 2 开发
