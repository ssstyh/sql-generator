# 员工信息与岗位职责

## 基本信息

- 岗位名称：前端工程师（Streamlit）
- 目录名称：`04_前端工程师_Streamlit`
- 岗位定位：负责交互界面、输入体验、结果展示与演示效果
- 项目阶段归属：Day 5 至 Day 9 核心岗位

## 在项目中的核心使命

将后端生成能力包装成一个清晰、专业、好上手的 Web 界面，让用户和面试官可以直接体验项目价值。

## 具体职责

### 1. 主界面开发

- 实现分析类型切换
- 实现 SQL 方言切换
- 根据不同分析类型切换动态表单
- 提供表名、字段名、日期范围、筛选条件等输入组件

### 2. 结果展示

- 在页面展示生成后的 SQL
- 多条 SQL 场景下使用 Tab 切换
- 提供复制到剪贴板功能
- 提供下载 `.sql` 文件功能

### 3. 交互增强

- 使用 `session_state` 保存历史记录
- 提供历史回看与清空历史按钮
- 提供欢迎说明和加载示例按钮
- 增加参数摘要卡片和高亮展示

### 4. 输入校验与体验优化

- 对非法输入做即时提示
- 非法状态下禁用生成按钮
- 保证页面结构简洁专业、适合演示
- 配合 demo 模式，让无数据库连接时也能体验功能

## 分阶段任务

- Day 5：完成主界面和核心交互
- Day 6：完成历史记录、示例加载、参数摘要、错误处理
- Day 7：配合测试修复交互问题
- Day 8 - Day 9：优化页面展示和上线体验

## 主要协作对象

- 技术负责人 / 后端架构师
- SQL 模板与分析逻辑工程师
- 测试工程师（QA）
- 部署与文档工程师

## 关键交付标准

- 页面可稳定切换不同分析类型
- 用户能完成输入、生成、复制、下载的完整流程
- 多 SQL 结果展示清晰
- 首次使用者无需讲解也能快速上手
