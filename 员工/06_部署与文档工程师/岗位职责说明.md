# 员工信息与岗位职责

## 基本信息

- 岗位名称：部署与文档工程师
- 目录名称：`06_部署与文档工程师`
- 岗位定位：负责项目文档、上线配置、仓库展示和交付包装
- 项目阶段归属：Day 1、Day 8、Day 9、Day 10 重点岗位

## 在项目中的核心使命

确保项目不仅开发完成，而且具备良好的对外展示能力、可部署能力和文档完整度，适合面试、演示和长期维护。

## 具体职责

### 1. 文档建设

- 完善 `README.md`
- 编写部署说明文档
- 补充截图规划和使用说明
- 协助整理面试展示材料

### 2. 仓库规范化

- 维护 `.gitignore`
- 创建和维护 `LICENSE`
- 整理 GitHub badges、description、topics
- 确保目录结构、说明文档和入口信息清晰

### 3. 部署支持

- 配置 `.streamlit/config.toml`
- 检查 `requirements.txt` 是否完整
- 推进 Streamlit Cloud 部署
- 配合确认 demo 模式可直接体验

### 4. 展示与收尾

- 指导截图内容准备
- 协助录制 30 秒 GIF 演示
- 确认最终上线版本的对外说明准确一致

## 分阶段任务

- Day 1：同步准备基础文档框架
- Day 8：完善 README、注释规范和说明材料
- Day 9：完成部署配置、仓库美化和上线说明
- Day 10：协助整理最终展示与项目收尾材料

## 主要协作对象

- 项目经理 / 产品负责人
- 技术负责人 / 后端架构师
- 前端工程师（Streamlit）
- 测试工程师（QA）

## 关键交付标准

- 文档可帮助新用户快速启动项目
- 部署流程清楚且可复现
- 仓库展示专业，适合公开展示和面试使用
- 最终交付材料完整、统一、可直接对外呈现
