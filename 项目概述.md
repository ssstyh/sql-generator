# SQL 分析模板生成器项目概述

## 项目简介

`SQL 分析模板生成器` 是一个 `CLI 或轻量 Web 工具`，用于根据用户输入的表名、字段名和分析参数，自动生成常用分析 SQL。

支持的分析场景包括：

- 日 / 周 / 月趋势分析
- 新增用户趋势
- N 日留存分析
- 漏斗转化分析
- RFM 用户分层
- 同比 / 环比分析

## 项目价值

在实际数据分析工作中，很多 SQL 分析模型写法相对固定，但每次换表都需要重写，效率较低。该工具的价值在于：

- 输入表结构和参数后自动生成标准 SQL
- 降低重复建模成本
- 帮助快速切换业务线
- 适合作为个人项目、工作提效工具和面试展示作品

一句话介绍：

> 把工作中常用的留存、漏斗、RFM、同环比等分析模型封装成工具，输入表结构即可自动生成可执行 SQL。

## 用户输入示例

```sql
-- 你只需要输入：
表名: user_orders
用户ID字段: user_id
时间字段: order_date
指标字段: amount
```

自动生成内容包括：

- 日 / 周 / 月活跃趋势
- 新增用户趋势
- N 日留存分析（次日、7日、30日）
- 用户分层（RFM）
- 同比环比对比
- 漏斗转化分析

- 技术栈：Python + Streamlit + Jinja2

实现留存分析 SQL 生成功能。

#### 输入参数

- `table_name`
- `user_id_field`
- `date_field`
- `retention_days`
- `start_date`

- `dialect`

2. 用 `LEFT JOIN` 匹配 N 天后是否仍有行为
3. 按首次活跃日期分组统计新增用户、留存人数和留存率

#### 技术要求

- 新建 `templates/retention.sql.j2`
- 在 `core/generator.py` 中实现 `RetentionGenerator`
- SQL 添加清晰中文注释
- 编写单元测试

#### 特别注意

- 留存 SQL 是高频面试考点
- 逻辑必须正确且便于解释

### 面试回答示例

#### 项目介绍

> 我在学习数据分析的过程中发现，留存、漏斗、RFM 这些经典分析模型的 SQL 写法相对固定，但每次换一张表就要重写。所以我做了一个工具，输入表名和字段就能自动生成标准 SQL。我用 Jinja2 做模板引擎，支持 MySQL 和 PostgreSQL 两种方言，前端用 Streamlit 做了交互界面。这个工具我自己学习和实习中一直在用。

#### 留存分析逻辑

> 先用子查询算出每个用户的首次活跃日期，然后用 LEFT JOIN 把原表关联回来，匹配 N 天后是否有行为记录，最后按首次活跃日期分组，计算每组的留存人数和留存率。我可以打开工具演示一下生成的 SQL。
