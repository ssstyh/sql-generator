Day 2：趋势分析 + 同比环比模板
给 Codex 的任务 2：

Markdown

在 sql-generator 项目中，实现趋势分析 SQL 生成功能。

## 输入参数
- table_name: 表名（字符串）
- date_field: 日期字段名
- metric_field: 指标字段名（如 amount）
- aggregation: 聚合方式（SUM / COUNT / COUNT DISTINCT / AVG）
- granularity: 时间粒度（日 / 周 / 月）
- user_id_field: 用户ID字段（可选，用于 COUNT DISTINCT）
- filter_conditions: 筛选条件（可选，列表形式）
- date_range: 日期范围（可选，起止日期）

## 需要生成的 SQL（共3个）

### SQL-1：基础趋势
按时间粒度聚合指标，输出日期 + 指标值

### SQL-2：同比对比
当期 vs 去年同期，输出日期 + 当期值 + 去年同期值 + 同比增长率

### SQL-3：环比对比
当期 vs 上一周期，输出日期 + 当期值 + 上期值 + 环比增长率

## 技术要求
- 使用 Jinja2 模板引擎，模板文件放在 templates/ 目录
- core/generator.py 中写一个 TrendGenerator 类
- 生成的 SQL 用 sqlparse 格式化
- 支持 MySQL 和 PostgreSQL 两种方言（日期函数不同）
- 每个生成的 SQL 顶部加注释说明（-- 分析类型：趋势分析 / 生成时间 / 参数说明）
- core/validator.py 中校验：表名不能为空、聚合方式必须合法

## 示例
输入：
  table_name = "user_orders"
  date_field = "order_date"  
  metric_field = "amount"
  aggregation = "SUM"
  granularity = "日"

期望输出（MySQL 方言）：
-- 分析类型：日趋势分析
-- 指标：SUM(amount)
-- 数据表：user_orders
-- 生成时间：2024-01-01 12:00:00

SELECT 
    DATE(order_date) AS dt,
    SUM(amount) AS metric_value
FROM user_orders
GROUP BY DATE(order_date)
ORDER BY dt;

Day 3：留存分析模板
给 Codex 的任务 3：

Markdown

在 sql-generator 项目中，实现留存分析 SQL 生成功能。

## 输入参数
- table_name: 用户行为表名
- user_id_field: 用户ID字段
- date_field: 行为日期字段
- retention_days: 留存天数列表（如 [1, 3, 7, 14, 30]）
- start_date / end_date: 分析日期范围
- dialect: MySQL 或 PostgreSQL
- filter_conditions: 可选筛选条件

## 需要生成的 SQL（共2个）

### SQL-1：经典 N日留存表
输出：首次活跃日期 | 新增用户数 | Day1留存数 | Day1留存率 | Day3留存数 | Day3留存率 ...

逻辑：
1. 先算每个用户的首次活跃日期
2. 再用 LEFT JOIN 匹配 N 天后是否有行为
3. 按首次活跃日期分组统计

### SQL-2：留存曲线数据
输出：留存天数 | 平均留存率
用于画留存衰减曲线

## 技术要求
- 新建 templates/retention.sql.j2
- core/generator.py 中新增 RetentionGenerator 类
- 生成的 SQL 加清晰注释
- 写单元测试

## 特别注意
- 留存 SQL 是面试高频考点，生成的 SQL 一定要逻辑正确
- 添加详细的中文注释，帮助用户理解每一步
